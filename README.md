# Перебор пароля RAR-архива (восстановление доступа к своим данным)

## Формулировка задачи и техническое задание

Требуется программа на Python, которая:

- Перебирает пароль к RAR-архиву из каталога `IN` (в конфиге задаётся имя файла, по умолчанию `Алекс пьян.rar`).
- Использует **как можно больше логических ядер CPU**, оставляя небольшой резерв, чтобы система не «замерзала».
- Ведёт учёт **всех проверенных паролей** во внешнем хранилище в каталоге `OUT` (SQLite), с полями: пароль, идентификатор архива, результат проверки, дата/время.
- При повторном запуске **не проверяет повторно** пары «этот архив + этот пароль», если они уже есть в базе.
- Порядок перебора **от простого к сложному**:
  1. **Числовой** перебор (только `numeric_charset`, длины из конфига).
  2. Для **каждого словаря по порядку** в `Config.json` — фаза **dictionary** (сначала маленький/простой список, в конце — крупный, например **xato-net-10-million-passwords.txt**).
  3. Для **каждого словаря в том же порядке** — фаза **hybrid** (слово + цифры/спецсимволы и комбинации); так самая тяжёлая гибридная фаза приходится на последний словарь.
- Поддерживается **несколько файлов-словарей** в одном конфиге (`wordlists.entries`), порядок в массиве = порядок прохода фаз.
- Способ проверки пароля задаётся в `Config.json`: **`rarfile`** (по умолчанию, под капотом всё равно нужен **UnRAR**) или прямой вызов **`unrar`** с путём и параметрами.
- Корректное завершение по **Ctrl+C** (на macOS в терминале это прерывание процесса) и по **Esc** (если установлен `pynput` и терминал получает события клавиатуры).
- Параметры перебора задаются в **`Config.json`**.

**Важно:** архивы в `IN` и **весь каталог `OUT`** перечислены в `.gitignore` и **не попадают в репозиторий** (ни SQLite, ни другие файлы внутри `OUT/`). Каталог `OUT` создаётся программой при запуске, если его ещё нет. Крупные словари (в том числе xato 10M) в `wordlists/*.txt` по умолчанию **не коммитятся**; в репозитории остаётся небольшой пример `wordlists/rockyou_sample.txt`.

### Параллелизм на CPU (проверено логикой программы)

- Число воркеров считается как **`os.cpu_count()`** (логические процессоры, с учётом гиперпотоков, если они есть) **минус** `parallelism.reserve_cpu_cores`, но не меньше **1**.
- Если задан **`parallelism.max_workers`**, берётся **минимум** из «все минус резерв» и этого лимита.
- Каждый воркер — **отдельный процесс** (`multiprocessing.Pool`, контекст **`spawn`**), внутри вызывается UnRAR/rarfile. Планировщик ОС распределяет процессы по ядрам; так задействуется **вся доступная CPU-ёмкость**, которую вы не «зарезервировали».
- При старте в **INFO-лог** выводится сводка: сколько логических CPU видит Python, резерв, лимит и **итоговое число процессов пула**.

### Про GPU, нейроускорители (NPU, Apple Neural Engine и т.п.)

Проверка пароля RAR через **rarfile** и **UnRAR** — **чисто CPU-задача**; эти библиотеки **не используют** видеокарту и не выгружают криптографию на NPU/«нейронные ядра».

В **`Config.json`** есть флаги **`parallelism.use_gpu`** и **`parallelism.use_neural_accelerator`** (по умолчанию **`false`**). Они **не включают** ускоритель в текущей реализации: это **зарезервированные переключатели** на будущее и напоминание в логе. Если один из них **`true`**, при запуске пишется **предупреждение**, что проверка по-прежнему идёт на CPU.

Реальный перебор RAR на GPU обычно делают **отдельными** средствами (**hashcat** + OpenCL/CUDA, подготовка хэша и т.д.), а не через этот пайплайн.

## Словарь xato-net-10-million-passwords.txt

Файл из набора **SecLists** (часто используют для «тяжёлой» фазы после маленького словаря). Сохраните его в каталог `wordlists` под именем **`xato-net-10-million-passwords.txt`** (как в `Config.json`).

Пример загрузки (из корня проекта):

```bash
curl -L -o wordlists/xato-net-10-million-passwords.txt \
  "https://raw.githubusercontent.com/danielmiessler/SecLists/master/Passwords/xato-net-10-million-passwords.txt"
```

Размер файла большой; убедитесь, что хватает места на диске. Для длинных строк в конфиге для этой записи задано увеличенное `max_line_length` (256).

## Установка и запуск

1. Создать и активировать виртуальное окружение (из корня проекта):

```bash
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

2. Установить **UnRAR** (для macOS часто: `brew install unrar`) либо указать полный путь в `Config.json` → `checker.unrar_path` или в `.env` как `UNRAR_PATH`.

3. (По желанию) скачать крупные словари в `wordlists/` согласно `Config.json`.

4. Запуск (любой из вариантов, из корня проекта с активированным `venv`):

```bash
python main.py
```

либо, как раньше:

```bash
export PYTHONPATH=src
python -m rar_bruteforce
```

Файл **`main.py`** сам добавляет каталог `src` в `sys.path`, переменная `PYTHONPATH` не обязательна.

При успехе в консоль выводится строка вида `УСПЕХ. Пароль: ...` и процесс завершается с кодом `0`.

## Описание решения

- **Конфигурация** читается из `Config.json`, опционально дополняется переменными окружения из `.env` (`python-dotenv`).
- **Порядок фаз** (`runner.build_phase_plan`):
  - `numeric` — декартово произведение символов из `password.numeric_charset` для длин от `min_length` до `max_length`.
  - Для каждого элемента `wordlists.entries` **по порядку**: `dictionary[label]` — только строки, длина которых попадает в **[`password.min_length`, `password.max_length`]**; более длинные строки в файле не перебираются (лимит чтения строки также не выше `max_length`).
  - Если `hybrid.enabled`: для **каждого** того же словаря — `hybrid[label]`; варианты «слово + цифры», «цифры + слово», спецсимволы и «слово + спец + цифра» **генерируются только если итоговая длина пароля в тех же пределах**; слова длиннее `max_length` из файла не используются как основа.
  - Перед проверкой пакет дополнительно фильтруется по SQLite: **уже проверенные для данного архива пароли не отправляются в воркеры**.
- **Обратная совместимость:** если в JSON остался старый блок **`wordlist`** (один путь) и нет **`wordlists`**, он автоматически превращается в список из одного словаря.
- **Параллелизм** — пул процессов `multiprocessing` с контекстом `spawn`. Число процессов: `os.cpu_count() - reserve_cpu_cores`, ограничивается `max_workers`, если задан. Подробный отчёт — `runner.log_parallelism_report`. Флаги `use_gpu` / `use_neural_accelerator` не меняют проверку пароля (только предупреждение в логе при `true`).
- **Проверка пароля** (`checker.py` + `worker.py`): режимы `rarfile` и `unrar`.
- **База попыток** (`db_sqlite.py`) — таблица `attempts` с уникальной парой `(archive_key, password)`.
- **Остановка** — SIGINT (Ctrl+C) и Esc (`pynput`). После текущего пакета данные пишутся в БД; в `finally` вызываются **`terminate()`** (если была остановка) или **`close()`** для пула, **`join()`**, проверка **`multiprocessing.active_children()`** с **`terminate`** для оставшихся дочерних процессов, остановка слушателя Esc. **Повторный Ctrl+C** — немедленный **`terminate`** активного пула.
- **Логи** — каталог `log/`. **Консоль и `INFO_*.log`:** уровень **INFO** и выше (ход работы, фазы, прогресс). **`DEBUG_*.log`:** имя файла по шаблону ТЗ; внутрь пишутся только **WARNING / ERROR / CRITICAL** (ошибки и предупреждения, в т.ч. первое появление каждого кода ошибки проверки пароля) в формате с `[class: … | def: …]`. Обычные успешные/неверные проверки пароля **не логируются** построчно.

## Переменные и функции (справочник)

### `Config.json` — блок `wordlists`

| Поле | Назначение |
|------|------------|
| `wordlists.defaults.encoding` | Кодировка строк в словарях по умолчанию |
| `wordlists.defaults.max_line_length` | Максимальная длина строки в файле (отсев слишком длинных строк) |
| `wordlists.defaults.skip_lines_starting_with` | Префикс комментариев (например `#`) |
| `wordlists.entries` | Упорядоченный массив словарей: **сначала простые, в конце сложные (xato)** |
| `entries[].path` | Путь к файлу относительно корня проекта или абсолютный |
| `entries[].label` | Метка для логов и имён фаз (`dictionary[label]`, `hybrid[label]`) |
| `entries[].encoding` | (Необязательно) переопределение кодировки для файла |
| `entries[].max_line_length` | (Необязательно) лимит длины строки для конкретного файла |
| `entries[].skip_lines_starting_with` | (Необязательно) префикс комментария для файла |

Элемент `entries` может быть **строкой** (только путь) — тогда `label` берётся из имени файла без расширения.

### Прочие поля `Config.json`

| Поле | Назначение |
|------|------------|
| `directories.in`, `directories.out` | Каталоги входа и выхода |
| `archive_file` | Имя RAR внутри `IN` |
| `database.filename` | Имя файла SQLite в `OUT` |
| `database.journal_mode` | Режим журнала SQLite (рекомендуется `WAL`) |
| `password.min_length`, `password.max_length` | Допустимая длина кандидата |
| `password.charset` | Полный алфавит (резерв для расширения сценариев) |
| `password.numeric_charset` | Символы для фазы «только цифры» |
| `hybrid.*` | Включение и параметры гибридной фазы |
| `checker.mode` | `rarfile` или `unrar` |
| `checker.unrar_path` | Путь к `unrar` (или `UNRAR_PATH` в `.env`) |
| `checker.unrar_command` | Подкоманда UnRAR, обычно `t` (тест) |
| `checker.unrar_timeout_sec` | Таймаут одного вызова `unrar` |
| `parallelism.max_workers` | Верхняя граница воркеров (`null` — все логические CPU минус резерв) |
| `parallelism.reserve_cpu_cores` | Сколько логических ядер не занимать воркерами (остаётся ОС и фон) |
| `parallelism.batch_size` | Размер пакета паролей между запросами к БД и пулу |
| `parallelism.use_gpu` | Зарезервировано: **`true` не включает GPU** для UnRAR; в лог пишется предупреждение |
| `parallelism.use_neural_accelerator` | Зарезервировано: **`true` не включает NPU/ANE**; в лог пишется предупреждение |
| `runtime.esc_key_enabled` | Включить прослушивание Esc |
| `runtime.progress_log_every_sec` | Периодичность прогресса в лог |

### Модули пакета `rar_bruteforce`

| Имя | Роль |
|-----|------|
| `config_loader.WordlistSpec` | Описание одного файла-словаря (путь, метка, кодировка, лимиты) |
| `config_loader.load_config` | Загрузка `AppConfig` из JSON и окружения |
| `runner.build_phase_plan` | Построение упорядоченного списка фаз (простое → сложное) |
| `generators.iter_numeric_passwords` | Числовая фаза |
| `generators.iter_dictionary_passwords` | Строки словаря для одного `WordlistSpec` |
| `generators.iter_hybrid_passwords` | Гибрид для одного `WordlistSpec` |
| `logging_setup.setup_logging` | Инициализация логов в `log/` (INFO в консоль и INFO-файл; диагностика в DEBUG-именованный файл) |
| `logging_setup.log_diagnostic` | Запись предупреждений/ошибок с полями class/def для файла диагностики |
| `db_sqlite.*` | SQLite: схема, фильтр уже проверенных, вставка |
| `worker.pool_initializer`, `try_password` | Воркер процесса |
| `runner.compute_worker_count` | Расчёт числа процессов пула по CPU и конфигу |
| `runner.log_parallelism_report` | Сводка в лог: CPU, резерв, воркеры, предупреждение про GPU/NPU |
| `runner.BruteForceRunner.run` | Полный цикл |
| `__main__.main` | Точка входа при `python -m rar_bruteforce`; вызывается из **`main.py`** в корне |

**Пример просмотра базы:**

```bash
sqlite3 OUT/checked_passwords.sqlite "SELECT password, result, checked_at FROM attempts ORDER BY id DESC LIMIT 20;"
```

## Ограничения и рекомендации

- Один архив ~1 ГБ и более: в режиме `rarfile` чтение **первого** элемента каталога может оказаться чтением очень большого файла и сильно замедлить каждую попытку. Имеет смысл переключить `checker.mode` на **`unrar`** либо использовать копию архива с маленьким тестовым файлом.
- Каждая попытка может нагружать диск; при перегреве или троттлинге уменьшайте `batch_size` или `max_workers` / увеличивайте `reserve_cpu_cores`.
- Фаза «только цифры» при длине до 8 — до **10⁸** комбинаций; оценивайте время.
- Фаза **hybrid** на словаре из 10 миллионов строк умножает число кандидатов; держите разумные `max_length` и параметры `hybrid`.
